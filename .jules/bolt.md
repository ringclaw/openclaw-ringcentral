## 2025-03-05 - Avoid regex replace inside hot loops for simple prefix checks
**Learning:** In `isSenderAllowed` inside `src/monitor.ts`, the `.replace` string method with a regular expression (`/^(ringcentral|rc):/i` and `/^user:/i`) was being invoked within a `some` loop on the `allowFrom` array. A standalone test (`benchmark.mjs`) showed that simple prefix checking via `.startsWith()` combined with `.substring()` is approximately 4-5x faster than using `.replace` with regex in loops. Additionally, an exact match early-return shortcut (`if (allowFrom.includes(normalizedSenderId)) return true;`) saves unnecessary string allocation.
**Action:** When validating string prefixes inside loops over an array, avoid regex `.replace` methods. Always favor simple string operations like `.startsWith` and exact match checks before diving into loop validation.
